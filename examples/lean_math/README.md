# lean_math — Lean 4 Theorem Proving via ShinkaEvolve

Evolves a Python program whose job is to supply correct **Lean 4 proof tactics**
for a fixed set of 15 mathematical theorems. The evolutionary loop uses LLMs to
improve the `get_proof_tactics()` function inside `initial.py`.

## Task design

| | |
|---|---|
| **Score** | Number of theorems that compile in Lean 4 (max **15**) |
| **Initial baseline** | 2 / 15 (`rfl` solves only definitional goals) |
| **Target** | 15 / 15 — all proofs compile, no `sorry` |

### Problem set (15 theorems)

| # | Name | Statement | Tier |
|---|------|-----------|------|
| 1 | `add_zero` | `n + 0 = n` | Basic |
| 2 | `zero_add` | `0 + n = n` | Basic |
| 3 | `mul_one` | `n * 1 = n` | Basic |
| 4 | `one_mul` | `1 * n = n` | Basic |
| 5 | `mul_zero` | `n * 0 = 0` | Basic |
| 6 | `add_comm` | `n + m = m + n` | Comm/Assoc |
| 7 | `add_assoc` | `(n+m)+k = n+(m+k)` | Comm/Assoc |
| 8 | `left_distrib` | `n*(m+k) = n*m + n*k` | Comm/Assoc |
| 9 | `zero_le` | `0 ≤ n` | Order |
| 10 | `lt_succ` | `n < n + 1` | Order |
| 11 | `le_add_right` | `n ≤ n + m` | Order |
| 12 | `and_intro` | `p ∧ q` from `p`, `q` | Logic |
| 13 | `and_flip` | `q ∧ p` from `p ∧ q` | Logic |
| 14 | `or_comm_thm` | `q ∨ p` from `p ∨ q` | Logic |
| 15 | `or_inl_thm` | `p ∨ False` from `p` | Logic |

## Files

```
examples/lean_math/
  initial.py             Seed program with EVOLVE-BLOCK (get_proof_tactics)
  evaluate.py            Shinka harness — runs Lean, scores proofs
  run_evo.py             Evolution runner (reads YAML config)
  shinka_small.yaml      30-generation test run
  shinka_continuous.yaml 1000-generation resumable run
  results/               Generated by evaluate.py / run_evo.py
```

## Prerequisites

1. **Lean 4** via elan (installed at `~/.elan/bin/lean`):
   ```bash
   export PATH="$HOME/.elan/bin:$PATH"
   lean --version   # Lean (version 4.29.0, ...)
   ```
2. **shinka-evolve** Python package:
   ```bash
   pip install shinka-evolve
   ```
3. **API key** for an LLM (Anthropic, OpenAI, Gemini, etc.) in `.env`:
   ```bash
   ANTHROPIC_API_KEY=sk-ant-...
   # or OPENAI_API_KEY=sk-...
   ```

## Quick start

```bash
cd examples/lean_math

# Test the baseline (no API key needed):
python initial.py            # should print score: 2/15
python evaluate.py           # full harness smoke-test

# Run a 30-generation evolution batch:
python run_evo.py

# Run continuous mode (1000 generations, auto-resumes):
python run_evo.py --config_path shinka_continuous.yaml

# Monitor with WebUI (in another terminal):
shinka_visualize --port 8888 --open
```

## Using the shinka_run agent CLI

```bash
shinka_run \
  --task-dir examples/lean_math \
  --results_dir results/lean_math_agent \
  --num_generations 50 \
  --set evo.llm_models='["claude-sonnet-4-6"]' \
  --set db.num_islands=2
```

## How evolution improves the score

The EVOLVE-BLOCK in `initial.py` contains `get_proof_tactics()` — a dict
mapping each problem name to a Lean 4 tactic string. The LLM is shown:

- The current program and its score
- The list of failed theorems and why they failed
- A detailed system prompt about valid Lean 4 tactics

The LLM then proposes a patch (diff or full rewrite) that swaps wrong tactics
(`rfl`) for correct ones (`simp`, `omega`, `exact Nat.add_comm n m`, etc.).
Each candidate is compiled by Lean — only those that produce zero errors score.

## Resuming runs

The `shinka_continuous.yaml` config uses a fixed `results_dir`. Re-running
`run_evo.py` with that config will detect the existing database and resume
from where it left off:

```bash
python run_evo.py --config_path shinka_continuous.yaml
```

## What the LLM needs to discover

| Problem | Correct tactic |
|---------|----------------|
| `zero_add`, `mul_one`, `one_mul` | `simp` |
| `add_comm` | `exact Nat.add_comm n m` or `omega` |
| `add_assoc` | `exact Nat.add_assoc n m k` or `omega` |
| `left_distrib` | `exact Nat.left_distrib n m k` |
| `zero_le` | `exact Nat.zero_le n` |
| `lt_succ` | `exact Nat.lt_succ_self n` |
| `le_add_right` | `exact Nat.le_add_right n m` |
| `and_intro` | `exact ⟨h1, h2⟩` |
| `and_flip` | `exact ⟨h.2, h.1⟩` |
| `or_comm_thm` | `exact Or.comm.mp h` |
| `or_inl_thm` | `exact Or.inl h` |
